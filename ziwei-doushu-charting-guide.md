# 紫微斗数排盘逻辑与实现说明

这份文档用于统一本项目的紫微斗数排盘口径，目标不是命理阐释，而是给程序实现提供一份可执行、可验证、可逐步扩展的规则说明。

当前建议分成两层理解：

- 规则层：先明确命宫、身宫、五行局、十四主星、辅星、四化、运限的推导顺序。
- 实现层：先落地一套稳定的 MVP 星曜集合，再逐步扩展到更多辅曜、杂曜、神煞。

## 1. 当前项目建议采用的统一口径

为了减少不同流派造成的结果偏差，当前项目先固定以下口径：

- 年界：先按农历正月初一换年，不先做立春换年。
- 月份：命盘排盘中的农历月、农历日直接来自公历转农历结果。
- 时辰：按十二时辰，不拆早子时/晚子时。
- 分钟：UI 输入支持 5 分钟步进，但排盘仍按所属时辰处理。
- 闰月：先保留闰月的原始月信息，不做“后半月并入下月”的特殊流派处理。
- 年龄：运限先按虚岁逻辑理解，大限起运年龄由五行局决定。
- 四化：先采用常见的“生年四化”口径。

如果后续要做成正式产品，以上口径都建议做成设置项。

## 2. 排盘输入与预处理

排盘最低输入项：

- 公历年、月、日
- 出生时、分
- 性别

程序预处理后至少要得到：

- 农历年、月、日
- 生年天干
- 生年地支
- 出生时支

后续命宫、身宫、五行局、主星、辅星、四化几乎都依赖这些基础量。

## 4. 本命盘推导顺序

建议严格按下面顺序实现：

1. 公历转农历，得农历月、农历日、生年干支、时支
2. 定命宫、身宫
3. 排十二宫宫职
4. 定命宫宫干
5. 定五行局
6. 起紫微星、天府星
7. 安十四主星
8. 安辅星、助曜
9. 定生年四化
10. 定大限、小限、流年

这个顺序很重要，因为后面的星曜经常依赖前一步结果。

## 5. 定命宫、身宫

当前项目采用常见口径：

- 寅宫起正月，顺数至生月
- 从该位置逆数生时，为命宫
- 从该位置顺数生时，为身宫

程序表达：

1. 以寅为正月起点
2. 顺数到出生农历月
3. 该点逆数时支，落点为命宫
4. 该点顺数时支，落点为身宫

这也是当前代码已经采用的口径。

## 6. 排十二宫宫职

命宫确定后，其余宫位按固定顺序依次排布：

命、兄弟、夫妻、子女、财帛、疾厄、迁移、交友、官禄、田宅、福德、父母

注意：

- “宫位所在地支”和“宫职”是两套概念。
- 盘面上某个格子是子宫、丑宫之类的地支位，同时也对应命宫、夫妻宫之类的宫职。

## 7. 定命宫宫干与五行局

### 7.1 五虎遁起寅宫天干

常用规则：

| 生年天干 | 寅宫起干 |
| --- | --- |
| 甲、己 | 丙寅 |
| 乙、庚 | 戊寅 |
| 丙、辛 | 庚寅 |
| 丁、壬 | 壬寅 |
| 戊、癸 | 甲寅 |

随后从寅宫顺行排出十二宫天干，落到命宫即可得命宫宫干。

### 7.2 五行局

当前项目采用“命宫宫干 + 命宫地支”推五行局的常见简化实现。

得到的五行局与起运年龄对应关系：

| 五行局 | 数值 | 起运年龄 |
| --- | --- | --- |
| 水二局 | 2 | 2 |
| 木三局 | 3 | 3 |
| 金四局 | 4 | 4 |
| 土五局 | 5 | 5 |
| 火六局 | 6 | 6 |

五行局至少影响两件事：

- 起紫微星
- 大限从几岁开始

## 8. 十四主星安星逻辑

### 8.1 起紫微星

先取：

- `d = 农历生日`
- `b = 五行局数值`，即 2/3/4/5/6

再按常见算法：

1. 找到最小的非负整数 `n`，使 `(d + n)` 可以被 `b` 整除
2. 令 `q = (d + n) / b`
3. 从寅宫起顺数 `q` 宫，得到基准位置
4. 若 `n = 0`，该宫即紫微星所在宫
5. 若 `n` 为偶数，则再顺行 `n` 宫
6. 若 `n` 为奇数，则再逆行 `n` 宫

### 8.2 起天府星

天府星不是重新独立计算，而是由紫微星位置映射得到。

当前项目已采用固定映射表实现这一步。

### 8.3 安十四主星

十四主星分成两组：

- 紫微系：紫微、天机、太阳、武曲、天同、廉贞
- 天府系：天府、太阴、贪狼、巨门、天相、天梁、七杀、破军

当前项目采用“相对位移”方式实现：

#### 紫微系相对紫微位移

| 星曜 | 相对紫微 |
| --- | --- |
| 紫微 | 0 |
| 天机 | -1 |
| 太阳 | -3 |
| 武曲 | -4 |
| 天同 | -5 |
| 廉贞 | -8 |

#### 天府系相对天府位移

| 星曜 | 相对天府 |
| --- | --- |
| 天府 | 0 |
| 太阴 | +1 |
| 贪狼 | +2 |
| 巨门 | +3 |
| 天相 | +4 |
| 天梁 | +5 |
| 七杀 | +6 |
| 破军 | +10 |

所有位移均按 12 宫取模。

## 9. 辅星与助曜的实现优先级

对当前 ESP32 小屏项目，不建议一口气全上，建议分层实现。

这样已经能覆盖本命盘中最常查看的一批核心星曜。

## 10. 第一阶段辅星安星规则

以下规则建议作为当前项目的第一阶段实现标准。

### 10.1 左辅、右弼

- 左辅：辰上起正月，顺数至生月
- 右弼：戌上起正月，逆数至生月

程序表达：

- 左辅 = 辰 + (农历月 - 1)
- 右弼 = 戌 - (农历月 - 1)

### 10.2 文昌、文曲

- 文曲：辰上起子时，顺数至生时
- 文昌：戌上起子时，逆数至生时

程序表达：

- 文曲 = 辰 + 时支序
- 文昌 = 戌 - 时支序

### 10.3 天魁、天钺

按生年天干落宫：

| 生年干 | 天魁 | 天钺 |
| --- | --- | --- |
| 甲、戊、庚 | 丑 | 未 |
| 乙、己 | 子 | 申 |
| 丙、丁 | 亥 | 酉 |
| 辛 | 午 | 寅 |
| 壬、癸 | 卯 | 巳 |

### 10.4 禄存、擎羊、陀罗

禄存按生年天干安星：

| 生年干 | 禄存 |
| --- | --- |
| 甲 | 寅 |
| 乙 | 卯 |
| 丙、戊 | 巳 |
| 丁、己 | 午 |
| 庚 | 申 |
| 辛 | 酉 |
| 壬 | 亥 |
| 癸 | 子 |

随后：

- 擎羊 = 禄存后一宫
- 陀罗 = 禄存前一宫

### 10.5 天马

按生年地支分组：

| 年支组 | 天马 |
| --- | --- |
| 寅、午、戌 | 申 |
| 申、子、辰 | 寅 |
| 巳、酉、丑 | 亥 |
| 亥、卯、未 | 巳 |

### 10.6 地空、地劫

常见口诀：

- 亥上子时顺安劫
- 逆回便是地空亡

程序表达建议：

- 地劫 = 亥 + 时支序
- 地空 = 亥 - 时支序

### 10.7 火星、铃星

按常见安星诀，可先按生年地支确定火星、铃星在“子时”的起点宫位，再顺数到出生时支。

常见分组如下：

| 年支组 | 火星起子时 | 铃星起子时 |
| --- | --- | --- |
| 申、子、辰 | 寅 | 戌 |
| 寅、午、戌 | 丑 | 卯 |
| 巳、酉、丑 | 卯 | 戌 |
| 亥、卯、未 | 酉 | 戌 |

程序表达建议：

- 火星 = 火星起点 + 时支序
- 铃星 = 铃星起点 + 时支序

所有结果都按 12 宫取模。

## 12. 运限逻辑

### 12.1 大限

大限常用口径：

- 一宫管十年
- 起运年龄取决于五行局
- 阳男阴女顺行，阴男阳女逆行

当前项目已采用这套口径。

### 12.2 小限

小限通常一岁一宫。  
这一块当前项目尚未完整实现，可以后补。

当前目标年份的天干地支确定后，可据此重新取流年四化，并根据流年命宫规则映射到本盘宫位。  
这一块当前项目也还在后续阶段。

## 13. 宫位详情页应该展示什么

用户点击某个宫位时，详情页不应只给一个“单主星”结论，而应显示该宫所有已纳入系统的星曜。

建议详情结构固定为：

1. 宫位标题  
   例如：`夫妻宫（酉宫）`
2. 标记信息  
   例如：是否为命宫、身宫
3. 大限信息  
   例如：`23-32 岁`
4. 同宫星曜总览  
   把该宫所有已经计算出的主星、辅星、助曜、四化对象先列一遍
5. 分组展示  
   主星、辅星/助曜、四化落宫分组列出

这样用户先看到“全量清单”，再看分类，不会误以为这个宫只有一颗主星。

## 14. 当前代码应对齐的数据模型

当前工程里的 `ziwei_chart_result_t` 已经具备以下结构：

- `major_star_palace[]`
- `minor_star_palace[]`
- `natal_transforms[]`

接下来建议对齐成下面的思路：

- `major_star_palace[]`：十四主星
- `minor_star_palace[]`：第一阶段辅星与助曜
- `natal_transforms[]`：生年四化对应到星曜

当前版本建议先把 `minor_star_palace[]` 扩为 12 颗：

- 左辅、右弼
- 文昌、文曲
- 天魁、天钺
- 禄存、擎羊、陀罗
- 天马、地空、地劫、火星、铃星

如果后续继续扩星，建议把结构演进成：

```c
typedef struct {
    const char *name;
    uint8_t category;
    int palace;
} ziwei_star_position_t;
```

再按数组统一收集，这样宫位详情可以直接遍历“所有已纳入星曜”，而不是分别写死主星循环、辅星循环、杂曜循环。

## 15. 当前项目的代码改造目标

基于上面的文档，当前代码层建议先做两件事：

### 15.1 文档对齐

- 保持命宫、身宫、五行局、十四主星、四化逻辑不变
- 明确“第一阶段已纳入星曜集合”
- 明确“宫位详情要显示全量星曜”

### 15.2 代码改造

- 在 `ziwei_chart` 中补入地空、地劫、火星、铃星
- 宫位详情增加“同宫星曜总览”
- 保留分组展示：主星 / 辅星与助曜 / 四化
- 盘面小格继续只显示摘要，不强行塞满全部星曜

这样既能保证 240x320 小屏可读性，也满足点开宫位后看全星曜的需求。
